Week 2: Common Vulnerabilities Deep Dive

Home > Week 2

Day 1: Cross-Site Scripting (XSS) Deep Dive

  • XSS Fundamentals: Understand what XSS is and why it's dangerous. Learn the three main types: Reflected, Stored/Persistent, and DOM-based.
  • Reflected XSS: Understand how it works, identify vulnerable parameters, and practice with basic payloads like <script>alert('XSS')</script>.
  • XSS Contexts: Learn about different contexts where XSS can occur (HTML, attribute, JavaScript, CSS).
  • Practice: Complete XSS challenges in PortSwigger Web Security Academy and OWASP Juice Shop.

Day 2: Advanced XSS & Filter Bypass

  • Stored XSS: Learn why it's more dangerous and practice finding it in comments, profiles, and forms.
  • DOM-based XSS: Understand JavaScript sinks and sources and practice identifying vulnerable code.
  • Filter Bypass Techniques: Study encoding (HTML, URL, JavaScript) and practice with payloads like <img src=x onerror=alert('XSS')> and <svg onload=alert('XSS')>.
  • Practice: Complete advanced XSS labs in PortSwigger.

Day 3: SQL Injection Fundamentals

  • SQLi Basics: Understand different SQLi types (error-based, union-based, boolean-based, time-based) and the classic ' OR '1'='1 payload.
  • Error-based & Union-based SQLi: Practice extracting database information through error messages and using UNION statements.
  • Database Enumeration: Learn to extract database, table, and column names.
  • Practice: Complete SQLi labs in PortSwigger and OWASP Juice Shop.

Day 4: Advanced SQL Injection & Blind SQLi

  • Blind SQL Injection: Understand boolean-based and time-based blind SQLi techniques.
  • Data Extraction: Practice extracting data character by character using functions like SUBSTRING() and ASCII().
  • Time-based Techniques: Learn to use functions like SLEEP() and WAITFOR DELAY.
  • SQLMap: Install and learn basic SQLMap commands for automated SQLi detection and exploitation.
  • Practice: Complete blind SQLi labs and use SQLMap on vulnerable applications.

Day 5: CSRF & IDOR Vulnerabilities

  • Cross-Site Request Forgery (CSRF): Understand how it works, the role of CSRF tokens, and different attack vectors. Practice creating PoCs.
  • Insecure Direct Object References (IDOR): Learn to identify predictable object references and practice horizontal and vertical privilege escalation.
  • Testing Techniques: Use Burp Suite Intruder for IDOR testing.
  • Practice: Complete CSRF and IDOR labs in PortSwigger.

Day 6-7: Weekend Practice & Vulnerability Integration

Day 6: Advanced Practice

  • Vulnerability Chaining: Learn to combine multiple vulnerabilities (e.g., CSRF + XSS) for greater impact.
  • Real-world Practice: Try more complex challenges in OWASP Juice Shop, VulnHub, or HackTheBox.
  • Bug Report Reading: Read 10-15 disclosed reports on HackerOne Hacktivity.

Day 7: Review & Preparation

  • Week 2 Review: Create cheat sheets for XSS and SQLi payloads.
  • Tool Mastery: Practice with Burp Suite Intruder and Comparer.
  • Prepare for Week 3: Read about reconnaissance techniques.
